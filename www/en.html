01Provable Simulation

A simulation used to make a decision has one question: can you trust it? On floating-point engines, the answer is fragile.

In the float world

The same scenario yields different results on a different CPU/compiler. The prediction drifts on its own; you can't tell whether a divergence is model error, a fault, or cyber-tampering — so you loosen the threshold and go blind. "The simulation ran" is a claim you can't prove.

In Berk DT

All math is fixed-point integer → the simulation side has zero-bit noise. The same input, the same state_hash on every machine. You compare the result in one line, reproduce it in one command, and catch tampering instantly.

Same setup + same input→ state_hash→ Windows ≡ Linux ≡ macOS ≡ browser

This claim is auto-verified by CI on three platforms on every change; see it live in the Hash Consensus and CPU ≡ GPU ≡ browser demos.

02What-If Engine — Deterministic Future Tree

A classic digital twin monitors and replays (what happened?). Berk DT goes one step further: it forks futures from a live state (what else could have happened?).

Live state (snapshot)→ fork N branches→ simulate each forward + score→ hash-addressed COAs

🌳 Advisor — decision-space exploration

Each branch runs under a distinct plan; the result state is scored and hashed. Same base + same plan → same result. The options (COAs) you produce are defensible and reproducible; the base state is never mutated.

🛰️ Shadow Twin — live verification

A deterministic twin runs beside the real system on the same inputs, tracking the residual between prediction and measurement and raising an alarm on the first frame of divergence. With zero simulation noise the threshold can be tight — a small wear/fault/injection trips it cleanly.

Advisor and Shadow Twin are the operational layer of the Sentinel edition; they share the same core as rollback netcode and the proof chain.

04Competition — at a glance

Determinism is now the goal of many engines; the difference is how it's achieved and what it comes with. Berk DT unites in one core the three things competitors hold separately.

Float engines · Rapier · Jolt · Box2D · Havok

Strong physics, but cross-platform determinism depends on float + compiler flags (fragile) and there's no decision/twin layer.

Photon Quantum · the only fixed-point peer

Fixed-point and deterministic, but Unity only + its own cloud; game-netcode focused, not a digital twin.

Berk DT fills the intersection: by-construction fixed-point determinism + an embedded decision/twin layer + formal verification + an operational layer (Shadow Twin · Advisor) — a single lightweight, embeddable core.
